2. Diseño Conceptual: Modelo Entidad/Relación

Unidad 2 - Modelo E/R y EER Extendido

Introducción

El Modelo Entidad/Relación (E/R) es una herramienta para el diseño conceptual de bases de datos. Permite representar de forma gráfica la estructura lógica de una base de datos.

Elementos del Modelo E/R

Entidades

Una entidad es un objeto del mundo real que se puede distinguir de otros objetos y del cual queremos almacenar información.

Atributos

Los atributos son las propiedades que describen a una entidad.

  • Simples: No se pueden dividir
  • Compuestos: Se pueden dividir en subatributos
  • Monovaluados: Un solo valor
  • Multivaluados: Varios valores
  • Derivados: Se calculan a partir de otros

Relaciones

Las relaciones representan asociaciones entre entidades.

Cardinalidad

  • 1:1 (Uno a Uno)
  • 1:N (Uno a Muchos)
  • N:M (Muchos a Muchos)

Participación

  • Total: Todas las instancias participan
  • Parcial: Algunas instancias pueden no participar

Modelo E/R Extendido (EER)

El modelo EER añade conceptos adicionales al modelo E/R básico:

Especialización y Generalización

  • Especialización: Proceso top-down de definir subgrupos
  • Generalización: Proceso bottom-up de agrupar entidades

Herencia

Las entidades especializadas heredan los atributos de la entidad general.

Restricciones

  • Disjunción: Una instancia pertenece a una sola subclase
  • Solapamiento: Una instancia puede pertenecer a varias subclases
  • Completitud total: Toda instancia debe pertenecer a alguna subclase
  • Completitud parcial: Puede haber instancias que no pertenezcan a ninguna subclase

Entidades Débiles

Una entidad débil es aquella que no puede existir sin otra entidad (entidad fuerte).
